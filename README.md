## Запуск проекта (Windows)

### Вариант 1. Через Git

```bat
git clone https://github.com/usrnmeee/LR-2-Semester
cd LR-2-Semester
scripts\setup_env.bat
```

### Вариант 2. ZIP + двойной клик

1. Скачайте ZIP с GitHub.
2. Распакуйте в любую папку.
3. Двойной клик на `scripts\setup_env.bat`.

# Извлечение данных (Extract)

Для загрузки данных используется скрипт `scripts/extract.py`, который читает параметры источника из конфигурационного файла и выполняет HTTP-запрос к API.

Пример запуска:

```bat
conda run -n data_env python scripts\extract.py configs\variant_09.yml
```

Скрипт:

* читает настройки из `/configs/variant_09.yml`;
* выполняет запрос к **World Bank API**;
* сохраняет исходный JSON-ответ в слой **raw**.

---

# Хранение Raw-данных

Raw-данные сохраняются в папку `/data/raw`.

Пример структуры:

```
data/
  raw/
    raw_YYYYMMDD_HHMMSS.json
```

Каждый запуск создаёт новый файл с временной меткой.
Сохраняется полный оригинальный ответ API для последующей обработки.

---

# Нормализация данных (Transform)

На этапе **Transform** raw JSON преобразуется в табличный формат.

Нормализация выполняется в ноутбуке:

```
notebooks/week3_eda.ipynb
```

Основные шаги:

1. загрузка raw JSON из `/data/raw`;
2. разворачивание структуры JSON в таблицу с помощью `pandas.json_normalize`;
3. переименование вложенных полей (`country`, `indicator`);
4. приведение типов данных (`date`, `value`);
5. удаление строк с отсутствующими значениями GDP;
6. удаление дубликатов.

---

Пример структуры:

Формат файла — **CSV**.

Каждый запуск нормализации создаёт новый файл с временной меткой.

# Структура Normalized-таблицы

Каждая строка таблицы соответствует:

**одному значению экономического индикатора для конкретной страны и года.**

Основные поля таблицы:

| Поле         | Тип    | Nullable | Источник в raw JSON | Описание                            |
| ------------ | ------ | -------- | ------------------- | ----------------------------------- |
| year         | int    | no       | `date`              | Год наблюдения                      |
| value        | float  | yes      | `value`             | Значение GDP в текущих долларах США |
| country_iso3 | string | no       | `countryiso3code`   | Трёхбуквенный ISO-код страны        |
| indicator    | string | no       | `indicator.id`      | Код экономического индикатора       |

# Витрина данных (Mart)

На этапе **Mart** normalized-данные преобразуются в агрегированную витрину, содержащую ключевые метрики (KPI).

В отличие от normalized-слоя, который хранит детализированные данные по годам, mart-слой представляет собой **готовые аналитические показатели**, удобные для использования в отчетах и дашбордах.

---

# Источники данных для витрины

Для построения витрины используются:

* normalized-данные:

```
data/normalized/normalized_YYYYMMDD_HHMMSS.csv
```

* справочник стран:

```
reference/countries.csv
```

Справочник используется для получения человекочитаемого названия страны (`country_name`).

---

# Обогащение данных (Join)

Перед расчётом метрик выполняется объединение:

```
normalized + reference → enriched data
```

Объединение выполняется по ключу:

```
country_iso3
```

После join проверяется:

* совпадает ли число строк до и после объединения;
* корректно ли подтянулись названия стран;
* отсутствуют ли дубликаты.

---

# KPI (ключевые метрики)

В витрине рассчитываются следующие показатели:

### 1. Последнее значение

* `latest_year` — последний доступный год
* `latest_value` — значение GDP за этот год

---

### 2. Изменение за 10 лет

```
delta_10y = value(year) - value(year - 10)
```

---

### 3. Среднее за последние 10 лет

```
avg_10y = среднее значение за последние 10 лет
```

---

### 4. Максимум и минимум

* `max_year`, `max_value` — год и значение максимума
* `min_year`, `min_value` — год и значение минимума

# Структура Mart-таблицы

Каждая строка витрины соответствует:

**набору KPI по временной серии данных.**

Основные поля:

| Поле         | Описание                      |
| ------------ | ----------------------------- |
| country_name | Название страны               |
| latest_year  | Последний доступный год       |
| latest_value | Значение GDP за последний год |
| delta_10y    | Изменение за 10 лет           |
| avg_10y      | Среднее за последние 10 лет   |
| max_year     | Год максимального значения    |
| max_value    | Максимальное значение         |
| min_year     | Год минимального значения     |
| min_value    | Минимальное значение          |

# Назначение слоёв

* **Raw** — хранение исходных данных API без изменений
* **Normalized** — табличное представление данных
* **Mart** — агрегированные метрики для анализа

Разделение на слои:

* упрощает поддержку проекта;
* позволяет переиспользовать данные;
* обеспечивает прозрачность обработки данных;
* соответствует реальной практике построения data pipeline.
