# Transform / Normalization Layer

Нормализация выполняется в исследовательском ноутбуке:

```
notebooks/week3_eda.ipynb
```
А также скриптом:
```
scripts/raw_to_normalized.py
```

Задача этого этапа — преобразовать raw JSON-ответ API World Bank в удобную табличную структуру для дальнейшего анализа.

---

## Input Data

Источник данных:

```
data/raw/variant_09/raw_YYYYMMDD_HHMMSS.json
```

Формат raw-ответа API:

```
[
  {metadata},
  [records]
]
```

где:

* первый элемент — метаданные ответа API;
* второй элемент — список наблюдений по годам.

Каждый элемент списка содержит:

* страну;
* экономический индикатор;
* год наблюдения;
* значение показателя.

---

## Data Transformation

Для преобразования структуры используется библиотека **pandas**.

Основные шаги трансформации:

1. **Загрузка raw JSON**

   JSON-файл читается из `/data/raw`.

2. **Разворачивание вложенной структуры**

   Используется:

   ```
   pandas.json_normalize()
   ```

   для преобразования вложенных полей (`country`, `indicator`) в плоскую таблицу.

3. **Переименование колонок**

   Вложенные поля приводятся к удобному формату:

   | Raw поле        | Normalized поле |
   |-----------------|-----------------|
   | indicator.id    | indicator       |
   | countryiso3code | country_iso3    |
   | date            | year            |

4. **Приведение типов данных**

   | Поле  | Тип      |
   | ----- | -------- |
   | date  | datetime |
   | value | float    |

5. **Очистка данных**

   Выполняются базовые операции очистки:

   * удаление строк без значения GDP (`value = NULL`);
   * удаление возможных дубликатов;
   * отбор только необходимых колонок.

# Mart Layer (Агрегированная витрина)

На этапе **Mart** normalized-данные преобразуются в агрегированную витрину, содержащую ключевые метрики (KPI) для анализа.

В отличие от normalized-слоя, который хранит детализированные данные (по годам), mart-слой содержит **агрегированные показатели**, готовые для использования в аналитике и дашбордах.

---

## Input Data

Источник данных для витрины:

```
data/normalized/variant_09/normalized_YYYYMMDD_HHMMSS.csv
```

Также используется справочник:

```
reference/countries.csv
```

Справочник содержит соответствие:

* `country_iso3` → `country_name`

---

## Data Enrichment (Join со справочником)

Перед расчётом метрик выполняется объединение normalized-данных со справочником стран:

```
df.merge(reference, on="country_iso3", how="left")
```

Цель:

* добавить человекочитаемое название страны (`country_name`);
* обеспечить удобство анализа.

После join выполняется проверка:

* совпадает ли число строк до и после объединения;
* отсутствуют ли дубликаты ключа;
* корректно ли подтянулись названия стран.

---

## Временная структура

Данные имеют **годовую гранулярность** (`year`).

Перед расчётами выполняется:

* сортировка по `year`;
* удаление строк с `NULL` значениями `value` для корректных вычислений KPI.

---

## Расчёт KPI

На основе normalized-данных рассчитываются ключевые метрики:

### 1. Последнее доступное значение

* `latest_year` — последний доступный год;
* `latest_value` — значение GDP в этот год.

---

### 2. Изменение за 10 лет

```
delta_10y = value(year) - value(year - 10)
```

Если данных за 10 лет назад нет — значение не рассчитывается (`NULL`).

---

### 3. Среднее за последние 10 лет

```
avg_10y = mean(value за последние 10 лет)
```

Используется диапазон:

```
[year_max - 9, year_max]
```

---

### 4. Максимум и минимум

* `max_year`, `max_value` — год и значение максимума;
* `min_year`, `min_value` — год и значение минимума.

Расчёт выполняется без учёта `NULL`.

## Обоснование разделения слоёв

Разделение на слои необходимо по следующим причинам:

* **Raw** — хранит оригинальные данные без изменений (для воспроизводимости);
* **Normalized** — обеспечивает удобную табличную структуру для обработки;
* **Mart** — содержит готовые метрики для анализа и визуализации.

Такое разделение:

* упрощает отладку пайплайна;
* позволяет переиспользовать данные;
* предотвращает смешивание логики обработки и аналитики;
* соответствует практике построения data pipeline в реальных проектах.
